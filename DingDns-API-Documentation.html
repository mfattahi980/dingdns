1. Overview & Authentication

DingDns exposes three categories of endpoints, each with a different authentication method:

CategoryBase PathAuth MethodUse Case
Public/health, /api/ipNoneHealth checks, IP detection
DDNS Update/api/ddns/update, /nic/updateToken query param or Basic AuthDynamic DNS updates from routers/clients
API Key Protected/api/*X-API-Key headerExternal applications (SaaS frontend, scripts)
Admin API/admin/api/*Bearer token in Authorization headerAdmin panel, management scripts

API Key Authentication

REQUEST HEADER
X-API-Key: your-api-key-here

Admin Bearer Token Authentication

REQUEST HEADER
Authorization: Bearer <token-from-login>
X-CSRF-Token: <same-token>
Admin tokens are obtained from POST /admin/api/login and expire after 24 hours. Include the token in both Authorization: Bearer and X-CSRF-Token headers.

Rate Limits

AreaLimit
API Key routes (/api/*)120 requests / minute
Admin routes (/admin/*)60 requests / minute

CORS

CORS allowed origins are controlled per API Key. If an API Key has allowed_origins: "*" all origins are permitted. You may restrict to specific domains in the Admin Panel under API Keys.

3. DDNS Update Endpoints

These endpoints allow routers, scripts, and clients to automatically update DNS A/AAAA records when the IP changes. Two protocols are supported: token-based (simple) and DynDNS-compatible (for routers).

GET /api/ddns/update Token Auth
Update a DNS record using a DDNS token. If ip is omitted, the caller's IP is used automatically.
Query ParamRequiredDescription
tokenrequiredDDNS token linked to the record
ip or myipoptionalNew IP address. Defaults to caller's IP if omitted
REQUEST EXAMPLE
GET /api/ddns/update?token=abc123xyz&ip=203.0.113.45
RESPONSE 200
{
  "success": true,
  "old_ip": "203.0.113.1",
  "new_ip": "203.0.113.45",
  "record_name": "home",
  "zone_name": "example.com",
  "changed": true
}
POST /api/ddns/update Token Auth
Same as GET version. Same query parameters apply.
GET /nic/update Basic Auth (DynDNS)
DynDNS-compatible endpoint. Compatible with most routers (ASUS, Mikrotik, pfSense, etc.). Use the DDNS token as the password in Basic Auth.
ParameterValue
UsernameAny string (ignored)
PasswordYour DDNS token
hostname query paramThe record hostname (required)
myip query paramNew IP (optional, defaults to caller IP)
ROUTER SETUP EXAMPLE (curl)
curl -u "user:YOUR_DDNS_TOKEN" \
  "https://api.dingdns.com/nic/update?hostname=home.example.com&myip=203.0.113.45"
RESPONSE (DynDNS format)
good 203.0.113.45     # IP was updated
nochg 203.0.113.45   # IP unchanged
badauth               # Invalid token

4.2 Records (API Key)

GET /api/records/zone/:id API Key
List all DNS records for a zone.
RESPONSE 200
{
  "records": [
    {
      "id": 1,
      "zone_id": 5,
      "name": "@",
      "type": "A",

      "ttl": 300,
      "priority": 0,
      "is_ddns": false,
      "created_at": "2026-05-11T12:00:00Z"
    }
  ],
  "total": 1
}
POST /api/records/zone/:id API Key
Create a new DNS record in a zone.
Body FieldRequiredTypeDescription
namerequiredstringRecord name. Use @ for root, * for wildcard
typerequiredstringRecord type: A, AAAA, CNAME, MX, TXT, NS, SRV, CAA, PTR
contentrequiredstringRecord value (IP for A, hostname for CNAME/MX, text for TXT)
ttloptionalintegerTime to live in seconds. Default: 300
priorityoptionalintegerPriority for MX and SRV records. Default: 0
REQUEST — A Record
{
  "name": "@",
  "type": "A",
  "content": "203.0.113.45",
  "ttl": 300
}
REQUEST — MX Record
{
  "name": "@",
  "type": "MX",
  "content": "mail.example.com",
  "ttl": 3600,
  "priority": 10
}
REQUEST — TXT Record (SPF)
{
  "name": "@",
  "type": "TXT",
  "content": "v=spf1 include:_spf.google.com ~all",
  "ttl": 3600
}
PUT /api/records/:id API Key
Update an existing DNS record. Only include fields you want to change.
Body FieldRequiredTypeDescription
nameoptionalstringNew record name
typeoptionalstringNew record type
contentoptionalstringNew record value
ttloptionalintegerNew TTL
priorityoptionalintegerNew priority
DELETE /api/records/:id API Key
Delete a DNS record and any associated DDNS tokens.

Complete Integration Example (PHP)

PHP — Full Flow: Login → Create Zone → Add A Record → Create DDNS Token
<?php
$base = 'https://api.dingdns.com';

// 1. Admin Login
$res = json_decode(file_get_contents("$base/admin/api/login", false, stream_context_create([
    'http' => ['method' => 'POST', 'header' => 'Content-Type: application/json',
               'content' => json_encode(['username' => 'admin', 'password' => 'yourpass'])]
])));
$token = $res->token;

// 2. Create Zone
$ch = curl_init("$base/admin/api/zones");
curl_setopt_array($ch, [
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_POST => true,
    CURLOPT_POSTFIELDS => json_encode(['name' => 'example.com']),
    CURLOPT_HTTPHEADER => ["Authorization: Bearer $token", "Content-Type: application/json",
                           "X-CSRF-Token: $token"]
]);
$zone = json_decode(curl_exec($ch));
$zoneId = $zone->id;

// 3. Create A Record
curl_setopt($ch, CURLOPT_URL, "$base/admin/api/records/zone/$zoneId");
curl_setopt($ch, CURLOPT_POSTFIELDS, json_encode([
    'name' => '@', 'type' => 'A', 'content' => '203.0.113.45', 'ttl' => 300
]));
$record = json_decode(curl_exec($ch));

// 4. Create DDNS Token for the record
curl_setopt($ch, CURLOPT_URL, "$base/admin/api/ddns-tokens");
curl_setopt($ch, CURLOPT_POSTFIELDS, json_encode([
    'record_id' => $record->id, 'label' => 'Home Router'
]));
$ddns = json_decode(curl_exec($ch));
echo "DDNS Update URL: $base/api/ddns/update?token={$ddns->token}\n";

// 5. DDNS Update (no auth needed, just the token)
file_get_contents("$base/api/ddns/update?token={$ddns->token}&ip=203.0.113.99");

  https://api.dingdns.com/api/records/zone/5
